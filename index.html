Docker là gì?

Docker là một nền tảng mã nguồn mở cho phép bạn tự động đóng gói ứng dụng và tất cả các thành phần phụ thuộc của nó (libraries, config, environment) vào một Container. Container này có thể hoạt động nhất quán trên bất kỳ hệ điều hành nào từ máy cá nhân (Local) đến máy chủ đám mây (Cloud).

Khởi động siêu tốc

Khởi động các dịch vụ chỉ trong vài giây thay vì vài phút như Máy ảo (VM).

Cô lập hoàn toàn

Tránh xung đột môi trường (Ví dụ chạy song song PHP 7.4 và PHP 8.2).

Tiết kiệm tài nguyên

Sử dụng chung nhân (Kernel) của hệ điều hành máy chủ, vô cùng gọn nhẹ.

Mô Hình Đối Tượng (Objects Model)

Nhấp vào một đối tượng dưới đây để xem cách chúng tương tác và giải thích chi tiết.

Image (Ảnh Tĩnh)

"Bản thiết kế" đóng gói ứng dụng

Container (Tiến Trình)

Một "thực thể chạy" từ Image

Volume (Ổ Đĩa)

Lưu trữ dữ liệu bền vững

Network (Mạng)

Kết nối các Container

Container là gì?

Nếu Image là bản thiết kế trên giấy, thì Container là ngôi nhà thực sự được xây dựng từ bản thiết kế đó. Nó hoạt động độc lập và hoàn toàn cô lập với hệ thống chủ cũng như các container khác.

Đặc tính kỹ thuật:

  • Có tiến trình (process) chạy độc lập, tài nguyên RAM/CPU được cấp phát cô lập.
  • Được tạo ra, khởi động, tạm dừng hoặc xóa đi bất cứ lúc nào một cách nhanh chóng.
  • Dữ liệu bên trong Container sẽ mất đi khi Container bị xóa (trừ khi dùng Volumes).
# Tạo & chạy một container:
docker run -d --name my-app -p 8080:80 nginx
Mẹo nhanh: Bạn có thể liên kết các Container lại với nhau trong một mạng cục bộ (Network) và đồng bộ tệp dữ liệu ra ngoài (Volume) để tránh mất dữ liệu.

Cấu Trúc File Dockerfile

Dockerfile là file văn bản chứa danh sách các chỉ thị để Docker Engine xây dựng nên một Docker Image tự động.

Di chuột vào từng dòng để xem giải thích
Dockerfile 

FROM node:18-alpine
LABEL maintainer="admin@docker.lab"
WORKDIR /usr/src/app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
ENV PORT=3000 NODE_ENV=production
EXPOSE 3000
VOLUME [ "/usr/src/app/data" ]
CMD [ "node", "server.js" ]

Giải Thích Từ Khóa

Hãy di chuột hoặc nhấp vào bất kỳ dòng nào trong Dockerfile bên trái để xem giải thích chi tiết về chỉ thị đó, cách hoạt động và lưu ý quan trọng khi lập trình thực tế.

Dockerfile Best Practices

  • Hãy luôn sử dụng các bản phân phối Linux gọn nhẹ như alpine để giảm tối đa kích thước Image.
  • Tận dụng cơ chế Caching Layer bằng cách chạy lệnh copy package cấu hình trước khi copy toàn bộ source code.
  • Đặt file .dockerignore để tránh đóng gói nhầm thư mục rác như node_modules hoặc .git.

Cấu Trúc Docker Compose (docker-compose.yml)

Docker Compose là một công cụ giúp định nghĩa và quản lý các ứng dụng đa container (Multi-container applications) chỉ qua một file YAML cấu hình duy nhất.

Cấu trúc Compose cơ bản

Di chuột hoặc nhấp vào các phần được đánh dấu trong file YAML để khám phá:

Services: Định nghĩa danh sách các container/dịch vụ cần chạy. Ở đây chúng ta có 2 dịch vụ: web (Webapp) và db (Cơ sở dữ liệu).
Ports: Áp dụng chuyển hướng (mapping) từ Port cổng ngoài hệ điều hành vật lý vào Port hoạt động bên trong Container.

Lệnh khởi chạy nhanh

Chạy toàn bộ cụm dịch vụ được định nghĩa trong file compose chỉ với 1 câu lệnh duy nhất:

docker compose up -d
docker-compose.yml 
version: '3.8'
services:
    web:
        build: .
        ports:
      - "80:3000"
        environment:
      - DB_HOST=db
        depends_on:
      - db
        networks:
      - app-network

    db:
        image: postgres:15-alpine
        volumes:
      - db-data:/var/lib/postgresql/data
        environment:
      - POSTGRES_PASSWORD=secret123
        networks:
      - app-network

volumes:
    db-data:

networks:
    app-network:
